# React 배수도 렌더링 안정화 검증 체크리스트

## 목적

렌더링 구조화와 성능 최적화 이후 편집모드, 시뮬레이션모드, 전체화면, 시나리오 저장 흐름에 회귀가 없는지 확인한다.

## 실행 전 기준

- 프론트엔드 서버: `http://127.0.0.1:5173`
- 백엔드 서버: `http://127.0.0.1:8000`
- 자동 검증:
  - `npm run lint`
  - `npm run build`

## 편집모드

- [ ] 앱 진입 후 편집모드가 정상 표시된다.
- [ ] 다크모드와 라이트모드에서 작업장 배경, 토지, 파이프, 라벨이 읽힌다.
- [ ] 시나리오 목록이 로드된다.
- [ ] 기본 시나리오를 선택하면 캔버스가 해당 JSON으로 갱신된다.
- [ ] 새 시나리오 버튼을 누르면 작성 UI가 열리고, 취소 시 원래 상태로 돌아온다.
- [ ] 시나리오 수정 모드에서 제목, 설명, 배수도 JSON을 저장할 수 있다.
- [ ] 수정 중 초기화를 누르면 선택한 시나리오의 마지막 저장 상태로 돌아간다.
- [ ] 단일 객체 선택 highlight가 명확하게 표시된다.
- [ ] 단일 객체 드래그가 부드럽게 움직이고 pointer up 후 위치가 저장된다.
- [ ] 다중 선택 후 드래그가 정상 동작한다.
- [ ] 파이프/시설/지형 리사이즈가 정상 동작한다.
- [ ] 포트 클릭 relation 생성이 정상 동작한다.
- [ ] relation detach 또는 삭제가 정상 동작한다.
- [ ] undo/redo가 드래그, 리사이즈, relation 변경 이후 정상 동작한다.
- [ ] JSON 내보내기가 현재 레이아웃을 기준으로 내려받아진다.
- [ ] JSON 불러오기가 현재 레이아웃을 교체한다.
- [ ] INP 다운로드 버튼이 정상 응답한다.
- [ ] 확대는 제한 없이 계속 증가하고, 텍스트 라벨이 이전보다 읽기 쉽다.
- [ ] 편집 정보 drawer가 열림/닫힘 상태에서 레이아웃을 덮지 않고 공간을 나눠 가진다.

## 시뮬레이션모드

- [ ] 시뮬레이션모드 진입 후 선택한 시나리오가 표시된다.
- [ ] 시나리오 목록이 편집모드와 같은 이름으로 표시된다.
- [ ] JSON 불러오기 버튼으로 레이아웃을 교체할 수 있다.
- [ ] 엔진 시작 버튼이 정상 응답한다.
- [ ] WebSocket 상태가 연결 상태로 바뀐다.
- [ ] tick 값이 증가하고 runtime badge가 갱신된다.
- [ ] 엔진 일시정지 버튼이 정상 응답한다.
- [ ] 엔진 재개 또는 다시 시작이 정상 응답한다.
- [ ] 엔진 정지 버튼이 정상 응답한다.
- [ ] 강수 슬라이더 조작 후 서버 제어 상태가 반영된다.
- [ ] 막힘 슬라이더는 드래그 중 즉시 서버 요청하지 않고 pointer up 시점에 반영된다.
- [ ] 선택한 파이프/시설 highlight가 주변 객체보다 명확하게 보인다.
- [ ] 토지 색상과 오수 라인 보더가 겹쳐 보이지 않는다.
- [ ] drawer 버튼이 가장 왼쪽에 있고, drawer가 열리면 drawer 상단 오른쪽으로 이동한다.
- [ ] drawer 열림/닫힘이 전체화면과 일반 화면 모두에서 정상 동작한다.
- [ ] 전체화면 진입 시 상단 페이지 헤더는 숨겨지고, 엔진 제어 바는 유지된다.
- [ ] 전체화면 종료가 정상 동작한다.
- [ ] 다크모드와 라이트모드에서 배경, 토지, 파이프, 라벨이 읽힌다.

## 성능 확인

- [ ] 편집모드에서 객체를 5초 이상 드래그해도 눈에 띄는 입력 지연이 없다.
- [ ] 편집모드에서 파이프/지형 리사이즈가 이전보다 부드럽다.
- [ ] 시뮬레이션 tick 수신 중 버튼과 슬라이더 입력 지연이 크지 않다.
- [ ] 강수 슬라이더 10초 조작 중 화면 animation이 크게 끊기지 않는다.
- [ ] 막힘 슬라이더 10초 조작 중 pointer up commit만 서버로 나간다.

## 회귀 발견 시 기록할 것

- 화면 모드: 편집모드, 시뮬레이션모드, 전체화면 중 하나
- 테마: 다크모드 또는 라이트모드
- 선택 시나리오 이름
- 재현 순서
- 기대 결과
- 실제 결과
- 브라우저 콘솔 오류
- 백엔드 콘솔 오류

## 안정화 완료 기준

- `npm run lint` 통과
- `npm run build` 통과
- 편집모드 핵심 조작 회귀 없음
- 시뮬레이션 엔진 시작/정지/WebSocket tick 회귀 없음
- 전체화면 drawer와 엔진 제어 바 회귀 없음
- 사용자가 체감 성능 개선 상태를 유지한다고 확인
